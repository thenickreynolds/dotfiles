# Review Changes

Review staged and unstaged changes before committing.

## Instructions

1. Show `git status -sb` for overview
2. Show `git diff --staged` for staged changes (what will be committed)
3. Show `git diff` for unstaged changes (what won't be committed)
4. Summarize the changes:
   - Files added/modified/deleted
   - Brief description of what changed
   - Any potential issues or things to note
5. Ask if user wants to proceed with commit, stage more files, or make changes
